# GameScript TODO

更新时间：2026-03-03

## P0

- [ ] 异常处理机制：补齐脚本级 `try/catch/finally` 与 `throw`
  - 目标：脚本可以在运行时错误后恢复，不再只能由 VM/宿主直接中断。
  - 验收：支持捕获运行时异常、重抛异常、`finally` 始终执行。

- [ ] 统一错误模型：定义可捕获的错误对象（类型、消息、堆栈）
  - 目标：脚本层可进行分类处理与日志归档。
  - 验收：错误对象在脚本侧可访问 `type/message/stack`（或等价字段）。

- [ ] 函数参数能力增强：默认参数、可选参数（后续可扩展命名参数/可变参数）
  - 目标：提高 API 可演进性，减少调用方改造成本。
  - 验收：函数定义和调用支持缺省参数值，缺省参数规则明确且有测试覆盖。

## P1

- [ ] 控制流补全：增加 `switch/match`（或等价分支机制）
  - 目标：降低复杂分发逻辑的嵌套深度，提升可读性。
  - 验收：覆盖字面量匹配、默认分支、分支跳转正确性。

- [ ] 集合与索引增强：切片（如 `a[i:j]`）、tuple/set 字面量、解构赋值
  - 目标：补齐脚本日常数据处理能力。
  - 验收：切片边界行为一致；解构支持常见 list/tuple 场景。

- [ ] 模块加载异常治理：导入失败/循环依赖/符号缺失可脚本侧处理
  - 目标：模块系统在大型工程中可控、可恢复。
  - 验收：导入类错误可以被 `try/catch` 捕获并给出可读错误信息。

## P2

- [ ] 语法糖补充：`+=` 等复合赋值、字符串插值、空值安全访问（如 `?.`）
  - 目标：减少样板代码，提升脚本编写效率。
  - 验收：语法与现有优先级规则一致，并有回归测试。

- [ ] 调试体验增强：断点/单步/变量查看、完善栈追踪与源码定位
  - 目标：降低排障成本，提高开发效率。
  - 验收：错误定位可映射到脚本文件与行列信息，支持最小调试闭环。

- [ ] 标准库完备化：文件/时间/随机/JSON/正则等基础能力补齐
  - 目标：减少对宿主 C++ 辅助函数的依赖。
  - 验收：提供稳定 API 与示例脚本，覆盖核心 IO 与文本处理场景。
